# Deep Reading v3.0 — 一键吃透一本书

> 发一本书的内容，自动走完全流程。不用记命令，不用手动操作。

## 核心理念

**极简输入，最大产出**：用户只需要发内容，系统自动完成所有分析。

## 工作模式

### 模式一：一键读书（推荐）

用户说"帮我读这本书"或发 `/read`，系统自动：

```
收到内容 → 全局扫描 → 智能标记章节 → 拆解核心章 → 出知识卡片
```

用户只需要在 2 个关键节点确认：
1. **扫描后确认**：系统问"这些是核心章节，要拆解吗？"
2. **卡片后确认**：系统问"还要继续深入吗？"

### 模式二：单步命令

用户也可以单独使用每个步骤（见下方命令列表）。

## 输出模式

根据使用场景自动选择：

| 模式 | 输出长度 | 适用场景 | 触发方式 |
|------|---------|---------|---------|
| **极简** | 3-5 行 | 手机端 | 手机端自动触发 |
| **标准** | 完整分析 | 电脑端 | 默认模式 |
| **研究** | 最详细 | 写论文/书评 | 用户说"详细分析" |

### 极简模式输出格式

```
📖 《书名》一句话：[核心洞见]

核心框架：[名称] → [步骤]
最反直觉：[观点]
立即行动：[具体方法]
```

### 标准模式

完整结构化分析（见各命令详情）。

### 研究模式

标准模式 + 批判性审视 + 跨书连接。

## 命令列表

| 命令 | 用途 | 一键模式中 |
|------|------|-----------|
| `/read` | 一键读书（全流程） | 主入口 |
| `/book-scan` | 仅全局扫描 | 自动执行 |
| `/book-skip` | 标记跳读章节 | 自动执行 |
| `/book-chapter` | 拆解指定章节 | 核心章自动执行 |
| `/book-quiz` | 质询检验 | 可选 |
| `/book-feynman` | 费曼输出 | 可选 |
| `/book-critique` | 批判审视 | 可选 |
| `/book-card` | 知识卡片 | 自动执行 |
| `/book-cross` | 跨书连接 | 读完多本可选 |
| `/book-progress` | 查看阅读进度 | 随时可查 |

## 设计原则

1. **原文锚定**：每个判断引用原文
2. **智能跳读**：自动区分骨架章和水章
3. **输出倒逼**：能讲明白才算懂
4. **进度可见**：随时知道读到哪了
5. **知识复利**：读完的书产生连接

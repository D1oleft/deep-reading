# 阅读进度

## 功能

查看当前书籍的阅读进度，支持继续阅读和批量管理。

## 输出格式（单本书）

```
📖 阅读进度：《书名》

开始日期：YYYY-MM-DD
阅读目的：[用户填写的目的]

✅ 全局扫描 [完成]
✅ 跳读标记 [完成]
✅ 第1章拆解 [完成]
✅ 第3章拆解 [完成]
⬜ 第5章拆解 [待做]
⬜ 知识卡片 [待做]
⬜ 笔记导出 [待做]

进度：3/6 步 (50%)

下一步：拆解第5章

继续阅读？（回复"继续"）
```

## 输出格式（多本书）

```
📚 阅读书籍列表

1. 《书名A》— 进度 50% — 2026-06-10 开始
2. 《书名B》— 进度 100% — 2026-06-15 完成
3. 《书名C》— 进度 25% — 2026-06-16 开始

要查看哪本？（回复数字）
```

## 进度记录方式

进度记录在项目目录的 `.reading-progress.json` 文件中：

```json
{
  "书名": {
    "started": "2026-06-16",
    "purpose": "解决XX问题",
    "steps": {
      "scan": "done",
      "skip": "done",
      "ch1": "done",
      "ch3": "done",
      "ch5": "pending",
      "card": "pending",
      "export": "pending"
    },
    "notes_file": "~/笔记/《书名》阅读笔记.md"
  }
}
```

## 自动恢复

当用户说"继续读《书名》"时：
1. 读取进度文件
2. 从上次中断的地方继续
3. 提示用户已完成和待做的步骤

## 进度统计

```
📊 阅读统计

本月已读：X 本
本月在读：Y 本
总阅读时长：Z 小时
最常阅读时间：[时间段]
```

## 用户输入
$ARGUMENTS

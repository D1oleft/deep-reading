# 一键读书 — 发内容，自动吃透

## 流程总览

```
用户发内容 → Step1扫描 → Step2跳读 → 用户确认 → Step3拆解 → Step4卡片 → 完成
```

## Step 1: 全局扫描（自动）

收到书籍内容后，立即执行全局扫描：

### 输出内容（精简版）
```
📖 《书名》扫描完成

核心问题：[一句话]
作者论点：[一句话]
全书结构：[3-5行树状图]

章节分析：
✅ 必读（骨架）：第1、3、5章
⚠️ 可略（证据）：第2、4章
❌ 可跳（延伸）：第6、7章

关键概念：[3个最重要的]
```

### 问用户：
> 核心章节是第1、3、5章。要拆解这些章节吗？（回复"继续"或指定章节）

## Step 2: 智能跳读标记（自动）

根据扫描结果，自动给每章打标签：

| 标签 | 含义 | 处理方式 |
|------|------|---------|
| 🔴 骨架 | 论证核心结构 | 必须拆解 |
| 🟡 证据 | 支撑材料 | 可略读摘要 |
| 🟢 延伸 | 扩展讨论 | 可跳过 |

### 跳读规则
- 如果用户时间 < 30分钟：只拆骨架章
- 如果用户时间 30-120分钟：拆骨架章 + 证据章摘要
- 如果用户时间不限：全部拆解

## Step 3: 核心章节拆解（自动）

对每个骨架章执行拆解，输出精简版：

```
📖 第X章：[标题]

论点：[一句话]
论证：[前提] → [推理] → [结论]
最有价值的一句话："原文"
隐藏假设：[内容]
一个没回答的问题：[内容]
```

### 每章拆解后自动问：
> 继续下一章？还是直接出卡片？

## Step 4: 知识卡片（自动）

所有核心章拆解完后，自动生成知识卡片：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📚 《书名》知识卡片
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

一句话：[核心洞见，能发朋友圈]

框架：[名称] → [1→2→3→4]

最反直觉：[观点 + 证据]

立即行动：
1. [具体方法]（第X章）
2. [具体方法]（第X章）
3. [具体方法]（第X章）

最大局限：[一句话]

金句："原文"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Step 5: 完成后

```
✅ 《书名》阅读完成

已分析：X/Y 章
跳过：Z 章（延伸内容）
知识卡片：已生成

还要继续吗？
1. 深入分析某章（回复章号）
2. 做批判性审视（回复"审视"）
3. 费曼检验（回复"检验"）
4. 完成（回复"完成"）
```

## 进度记录

每完成一步，自动记录进度：

```
📖 阅读进度：《书名》

✅ 全局扫描 [完成]
✅ 跳读标记 [完成]
✅ 第1章拆解 [完成]
✅ 第3章拆解 [完成]
⬜ 第5章拆解 [待做]
⬜ 知识卡片 [待做]

进度：3/5 步
```

## 边界情况

### 用户只发了书名（没发内容）
> 请发送书籍内容（目录、前言或全文）。如果有电子文件，发文件路径也行。

### 用户发的内容太短（< 500字）
> 内容太少，无法做深度分析。请发送更多内容（至少目录和前言）。

### 用户中途退出
下次用户说"继续读《书名》"时，自动加载进度，从上次中断的地方继续。

### 用户要求速读
> 速读模式：只扫描 + 出卡片，约10分钟。
